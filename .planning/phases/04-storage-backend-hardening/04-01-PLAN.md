The `persist` middleware's `storage` option accepts a `PersistStorage<S>` or, more commonly via `createJSONStorage`, a `StateStorage`:

```typescript
// Zustand v5's StateStorage shape (https://zustand.docs.pmnd.rs/integrations/persisting-store-data#storage)
interface StateStorage {
  getItem: (name: string) => string | null | Promise<string | null>;
  setItem: (name: string, value: string) => unknown | Promise<unknown>;
  removeItem: (name: string) => unknown | Promise<unknown>;
}
```

From `idb-keyval` v6 (canonical async key-value API, ~600 LOC, no deps):
```typescript
export function get<T = any>(key: IDBValidKey): Promise<T | undefined>;
export function set(key: IDBValidKey, value: any): Promise<void>;
export function del(key: IDBValidKey): Promise<void>;
// (plus createStore, getMany, setMany, etc тАФ not needed here)
```
