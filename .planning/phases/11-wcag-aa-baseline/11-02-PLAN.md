<objective>
Two thin a11y wins shipped together because both touch shell-level code at the same wave and have no file overlap.

**Part A — A11Y-02: Make Toast announce to screen readers.**
`src/components/ui/Toast.tsx` is the canonical save-confirmation + milestone surface for the diary. Today the root `<div>` has no ARIA semantics — every "Pee saved", "Drink saved", "Day 1 complete!", "+250 mL" toast is invisible to NVDA / VoiceOver / TalkBack users. Fix: add `role="status"` + `aria-live="polite"` + `aria-atomic="true"` to the visible toast root. `role="status"` is the WCAG 4.1.3 Status Messages pattern for non-urgent confirmations; the audit (UI-REVIEW.md §C4 + finding I4 in 11-CONTEXT.md) calls out toasts as the prime miss.

There is currently only one Toast component used app-wide. All callers (DayPageClient.tsx, others) pass non-urgent messages. We are NOT adding a `role="alert"` variant prop in this plan because:
- The audit explicitly enumerates "milestone toasts and time warnings" (both non-urgent → `role="status"`) as the present-day gap.
- No live caller emits an "error toast" through `Toast.tsx` today (export errors went through `alert()` until STAB-07; the STAB-07 follow-up uses the Toast component for export errors too — those are still appropriate for `role="status"` since they ARE confirmations, not interruptions; the existing "Pee saved" baseline pattern stays correct).
- Adding a prop API for "polite vs assertive" is scope creep into v2. Phase 11 makes existing toasts audible; if a future phase introduces a high-urgency toast variant, it can add the prop then.

**Part B — A11Y-03: Skip-to-content link.**
WCAG 2.4.1 (Bypass Blocks, Level A) requires keyboard users be able to skip past repeated nav blocks. Right now a keyboard user landing on `/diary/day/1` must Tab through the header (logo, top-nav links, Learn, language switcher) before reaching the page's primary action. Fix: add a visually-hidden-until-focused anchor as the first focusable element inside `AppShell.tsx`, pointing at `#main-content`. The existing `<main>` element inside AppShell.tsx gains `id="main-content"`.

i18n: a new `messages/en.json` key `nav.skipToContent` ("Skip to content") replaces a hardcoded string. The `i18n-sync` PostToolUse hook (per CLAUDE.md, registered in `.claude/settings.json`) automatically mirrors the key into all 5 non-EN locale JSON files when the en.json save fires. Per memory `project_translation_workflow`: this is enforced by the article-i18n-completeness Stop hook + pre-commit hook (those gates apply to MDX articles, not UI strings — UI strings are mirrored by the `i18n-sync` skill). The executor verifies the mirror landed by reading messages/fr.json (etc.) after en.json saves and confirming `nav.skipToContent` is present in every locale file.

**Boomer-safe UX constraint (CLAUDE.md + UX_PHILOSOPHY.md):** the skip link is invisible to mouse users on the visible page — `sr-only` (Tailwind's built-in utility, ships with Tailwind 4 in this project) — and only visible during keyboard focus (`focus:not-sr-only`). It does NOT clutter the visible page chrome. This preserves the boomer-safe constraint per the audit's "skip-link must be invisible until focused" rule.

Output: 3 files touched, ~10 lines total. No visual regression. Phase 11 Wave 1.
