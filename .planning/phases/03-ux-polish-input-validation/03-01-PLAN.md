<objective>
Close ROADMAP §"Phase 3" success criterion #1: triggering the first-void
milestone toast on `/en/diary/day/1`, then switching to `/fr/diary/day/1`
mid-session, MUST NOT re-fire the toast in the new locale.

Per CONTEXT 03-CONTEXT.md, the existing `checkMilestone()` at
`DayPageClient.tsx:25-32` keys its `sessionStorage` flag as
`milestone_${key}` where `key` is locale-independent (`'first_event'`,
`'day1_complete'`, etc.). So the dedup *should* already work. The plan
honors a **reproduction-first** strategy: write the regression test
FIRST, run it against the current source, and only patch the source if
the test fails. Either outcome ships a permanent regression guard.

Per CONTEXT, this is an atomic Wave-1 plan with NO file overlap with
03-02 or 03-03. Worktree-isolated for parallel execution.
</objective>

**Final test set required by this plan (write exactly these three):**

1. `it('checkMilestone-equivalent: sessionStorage key milestone_first_event is locale-independent')`
   — Set `sessionStorage.setItem('milestone_first_event', '1')` BEFORE
   any render. Render `<DayPageClient/>` with `NextIntlClientProvider
   locale="en"`. Unmount. Re-render with `locale="fr"`. Assert
   `sessionStorage.getItem('milestone_first_event') === '1'` still.
   This proves: across locale switch, the dedup flag is preserved.
2. `it('after first save under EN, switching to FR does not re-fire the milestone toast')`
   — Driver: call `useDiaryStore.getState().addVoid(...)` to seed one
   void; then directly set `sessionStorage.setItem('milestone_first_event', '1')`
   to simulate the post-fire state. Mount under FR. Wait 1500ms
   (`vi.useFakeTimers()` + `vi.advanceTimersByTime(1500)`). Assert NO
   toast element is in the DOM (`screen.queryByText(frMessages.milestones.firstEvent)`
   returns null). This is the regression guard.
3. `it('fresh session under FR — first save DOES fire the FR milestone toast')`
   — Negative-control: clear sessionStorage, mount under FR, no
   precondition. (This test only exists to prove the test infra CAN
   detect a milestone fire; without it, test #2 could pass by virtue
   of the test never being able to detect a toast at all.) The way
   to drive a save in this test: directly mutate
