### Task 2 — Render the Toast at the bottom of the component's JSX

At the end of the returned `<div>` (after the `{!hasData && ...}`
block at line 137 and BEFORE the closing `</div>` at line 138),
insert:

```tsx
<Toast
  message={errorToast ?? ''}
  visible={errorToast !== null}
  onDismiss={() => setErrorToast(null)}
  duration={5000}
/>
```

Add the import at the top of the file (alongside the existing
`import Button from '@/components/ui/Button';` at line 5):

```tsx
import Toast from '@/components/ui/Toast';
```

Rationale for the wiring:
- `message={errorToast ?? ''}` — Toast requires `message: string`
  (non-nullable). The empty-string fallback is unreachable in practice
  because `visible={errorToast !== null}` gates rendering, and `Toast`
  itself early-returns `null` when `visible` is false. The `?? ''`
  satisfies TypeScript without runtime impact.
- `duration={5000}` — CONTEXT locked decision: error toasts are
  longer than save toasts (3000ms) because errors need more reading time.
- `onDismiss={() => setErrorToast(null)}` — single source of truth;
  fires from both the 5s timer (Toast.tsx:17-19) AND the close-X click
  (Toast.tsx:39-45).

Do NOT pass `emoji` or `subtitle` — error toasts use the default
`CheckCircle2` icon... wait, that's a success icon. **Read Toast.tsx
line 28-32 again:** when `emoji` is undefined, the icon falls back to
`<CheckCircle2 className="text-success" />` — visually a green checkmark.
That's wrong for an error toast.

**HANDLE THIS:** to avoid a green-checkmark-on-error visual bug,
pass an emoji that reads as a problem indicator. Use a single
emoji literal — NO new i18n key, this is a visual symbol, not text:

```tsx
<Toast
  message={errorToast ?? ''}
  emoji="⚠️"  // ⚠ warning-sign U+26A0 + variation-selector-16
  visible={errorToast !== null}
  onDismiss={() => setErrorToast(null)}
  duration={5000}
/>
```

The `⚠️` Unicode escape (warning sign + emoji-presentation
selector) renders consistently across iOS, Android, and desktop
browsers and matches the existing emoji-as-string pattern from
`DayPageClient.tsx:61-64` (e.g., `'\u{1F4AA}'`).
