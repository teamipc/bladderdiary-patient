# Plan 05-03: Hide BottomNav at md+, retire AppShell pb-24 spacer at md+, give Footer desktop padding

<objective>
Three small, surgically-scoped chrome changes that together flip the AppShell layout into "desktop mode" at the `md` (768px) breakpoint while preserving every byte of mobile (<768px) class strings:

1. **`BottomNav.tsx`** — Add `md:hidden` to the outer fixed `<nav>`. The nav stops rendering at viewport ≥768px (CSS-only; no React conditional, no `useBreakpoint` hook per UI-SPEC §"Open Questions" Q8 lock). Below 768px the nav, scroll-hide behavior, locked-Diary indicator, and Track-disabled state ALL render byte-for-byte identically.

2. **`AppShell.tsx`** — Change the `aria-hidden` spacer from `pb-24` to `pb-24 md:pb-0`. At <768px the existing 96px bottom space is preserved (so content does not slip under the still-visible BottomNav). At ≥768px the spacer collapses to zero because the BottomNav is no longer there.

3. **`Footer.tsx`** — Replace the inner wrapper `<div className="max-w-2xl mx-auto px-4 sm:px-6 py-10 text-center">` with `<Container variant="narrow" as="div" className="py-10 md:py-12 lg:py-16 text-center">`. The geometry contract (`max-w-2xl mx-auto px-4 sm:px-6` — exactly the existing class set) is preserved by the `narrow` variant; the new `md:py-12 lg:py-16` adds desktop-appropriate vertical padding (UI-SPEC §"6. `Footer` (MODIFIED — desktop padding only)").

These three changes have ZERO file overlap with plans 05-04, 05-05, 05-06 — they can run in parallel within Wave 2 (and 05-06 in Wave 3 against 05-01) without conflict. They depend ONLY on Container existing (Plan 05-01).

Mobile invariants (CRITICAL — UI-SPEC §"Mobile Invariants"):
- BottomNav at <768px: identical class strings (the only delta is `md:hidden` which is `display:none` only at ≥768px and inert below)
- AppShell `pb-24` at <768px: preserved (the `md:pb-0` only takes effect at ≥768px)
- Footer at <768px: padding stays `py-10` (the `md:py-12 lg:py-16` only takes effect at ≥768px); `max-w-2xl` and `mx-auto` and `px-4 sm:px-6` all preserved by `Container variant="narrow"`

RTL invariants (UI-SPEC §"Phase 5 RTL audit"):
- BottomNav: only adds `md:hidden` — direction-neutral. Existing logical `-end-0.5` on locked diary (line 126) preserved.
- AppShell: only modifies `pb-24` (block-axis property — direction-neutral).
- Footer: `Container variant="narrow"` uses only direction-neutral classes (`mx-auto`, `max-w-2xl`, `px-*`); existing inner `dir="ltr"` on the email link preserved.
- NO `ml-*`/`mr-*`/`pl-*`/`pr-*`/`left-*`/`right-*` introduced in any of these three files.

Open questions addressed:
- Q5 (`Button.tsx` focus-visible migration) — NOT in this plan. UI-SPEC locks it as Phase 6.
- Q7 (auto-hide-on-scroll listener at desktop) — NOT in this plan. UI-SPEC §"Note on the auto-hide scroll listener" defers it; the listener still runs at desktop but does no rendering work because the nav is `display:none`.
</objective>
