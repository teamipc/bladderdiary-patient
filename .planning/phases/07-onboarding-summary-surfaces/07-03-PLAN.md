# Plan 07-03: ExportActions.tsx — desktop hover affordance (md:hover:-translate-y-px + md:duration-150) + md:max-w-2xl mx-auto responsive wrap

<objective>
Apply the locked Phase 7 design contract from `07-UI-SPEC.md` §"Export Actions" to `src/components/export/ExportActions.tsx` (139 lines). Single file modification; no other surfaces touched (file-ownership rule — Plan 07-01 owns OnboardingFlow, Plan 07-02 owns summary).

This plan closes the export-button hover affordance + responsive wrap half of DTUX-05 by:
1. Adding `md:hover:-translate-y-px md:transition-all md:duration-150` to BOTH the PDF (primary) Button at line 109 and the CSV (secondary) Button at line 121. The `-translate-y-px` is the MAX 1px lift per Boomer-safe override 2; the `duration-150` is under the 200ms cap per Boomer-safe override 3. The hover-bg-color shift is INHERITED from the Button primitive's variant hovers (`primary` already hovers `bg-ipc-500 → bg-ipc-600`; `secondary` already hovers `bg-ipc-50 → bg-ipc-100`) — Phase 7 does NOT override these per UI-SPEC §"Why NO color shift to ipc-50 (deviating from CONTEXT spec)" — Tailwind's last-class-wins resolution would have any `hover:bg-ipc-50` we added defeat the primitive's `hover:bg-ipc-600`, which would visually un-fill the primary CTA on hover (wrong).
2. Adding `md:max-w-2xl md:mx-auto` to the outer wrapper `<div>` at line 107-108. The wrapper currently is `<div className="space-y-3">`; it becomes `<div className="space-y-3 md:max-w-2xl md:mx-auto">`. Effect at md+ inside the summary page's `Container variant="default"` (max-w-3xl = 768px): the export buttons row caps at 672px and centers, instead of stretching to ~720px of the Container's inner width. Subtle but visually settling. `max-w-2xl` matches Phase 5's `narrow` Container variant for design-system consistency, AND it has enough width to comfortably fit long-translation locales (PT, AR) without label wrap on the multi-word labels ("Send PDF to your healthcare team", etc.).
3. Focus-visible rings are INHERITED from the Button primitive (Phase 6 work). NO additional `className` work needed for focus per UI-SPEC §"Focus-visible spec — INHERITED from Button primitive".

**CRITICAL — what this plan does NOT do (UI-SPEC verified):**
- Does NOT add `hover:bg-ipc-50` or `hover:text-ipc-700` className overrides on either Button. UI-SPEC §"Why NO color shift to ipc-50 (deviating from CONTEXT spec)" — the Button primitive's variant hovers already own the bg-color animation; layering a className `hover:bg-ipc-*` on top would CONFLICT (last-class-wins would have `ipc-50` defeat `ipc-600` on the primary, which would un-fill the CTA on hover — wrong). The 1px lift is the NEW signal Phase 7 layers; the bg-color shift is already in the primitive.
- Does NOT change the Button `size` (stays default `size="lg"` inferred = min-h-[52px], above 44px floor) or `variant` (`primary` for PDF, `secondary` for CSV).
- Does NOT change the `fullWidth` prop or the icon rendering or the conditional `pdfOnly` / `hasData` / `shareSupported` branches. UI-SPEC §"Mobile invariants for ExportActions" verified all logic preserved — Phase 7 is style-only.
- Does NOT change the `animate-cta-shimmer` className when `shimmer` prop is true. UI-SPEC §"Mobile invariants for ExportActions" — shimmer fires at all viewports per existing intent.
- Does NOT change the "no data" empty-state message at line 132-136. It IS however centered inside the new `md:max-w-2xl md:mx-auto` wrap as a side effect (consistent positioning with the buttons it sits below).
- Does NOT change the `<div className="space-y-3">` to a different layout primitive. The wrapper stays a `<div>` with the existing `space-y-3` plus the new responsive `md:max-w-2xl md:mx-auto` tokens appended.
- Does NOT add any new i18n strings. UI-SPEC §"i18n + RTL Contract" verified: ZERO new keys; all `export.*` translation strings already exist in messages/*.json.
- Does NOT remove the existing `alert(t('pdfError', ...))` or `alert(t('csvError'))` calls at lines 68 + 94. STAB-07 (Phase 3 of Stabilization) is the separate work that swaps `alert()` for Toast — explicitly OUT of Phase 7 scope.
