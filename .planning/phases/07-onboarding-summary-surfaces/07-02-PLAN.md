# Plan 07-02: summary/page.tsx desktop typography + 3-stat tile padding bump (NO multi-column metric grid — IPC metrics live in CSV/PDF exports only)

<objective>
Apply the locked Phase 7 design contract from `07-UI-SPEC.md` §"Summary Page Layout" to `src/app/[locale]/summary/page.tsx` (328 lines). Single file modification; no other surfaces touched (file-ownership rule — Plan 07-01 owns OnboardingFlow, Plan 07-03 owns ExportActions).

This plan closes the typography + layout half of DTUX-05 (the export-button hover half is Plan 07-03) by:
1. Bumping H1 (line 114) from `text-2xl` to `text-2xl md:text-4xl` + removing the mobile-only `px-4` gutter at md+ via `md:px-0`. The H1 jump (24px → 36px) is the biggest typography bump in Phase 7 — the summary celebration is the moment the patient earns the visual congratulations per Design DNA axis 1 + UX_PHILOSOPHY behavioral lever L1 ("loss aversion / sunk-cost framing — the user always sees what they have already invested").
2. Bumping the two section H2s (storyTitle line 221, lookBackTitle line 253) from `text-lg` to `text-lg md:text-xl`. Small 18 to 20px bump preserves H1:H2 ratio at ~1.8x — confident hierarchy without shouting.
3. Bumping the 3-stat top grid tile padding (lines 138, 144, 150) from `px-2 py-3` to `px-2 py-3 md:px-4 md:py-5`. Each tile gets proportionally more interior whitespace at md+ where the tile rendered width grows from ~115px (375px viewport) to ~240px (768px+ inside max-w-3xl).

**CRITICAL — what this plan does NOT do (UI-SPEC verified):**
- Does NOT widen Container variant from `default` (max-w-3xl) to `wide` (max-w-5xl). UI-SPEC §"Container variant decision" + §"Q3" verified: the 5 IPC clinical metrics (24HV/NPi/AVV/MVV/NBC) are NOT rendered in-page — they live ONLY in `exportCsv.ts` + `exportPdf/*` for the artifacts the patient ships to the clinician. There is no in-page block that would benefit from a 5-up grid at lg+. The CONTEXT.md assumption about a multi-column metric grid was based on a misread of source; UI-SPEC corrected it. Widening to `wide` would stretch long-form prose ("storyBody", "comingBackBody") to 1024px which exceeds optimal reading line length (50-80ch = 600-800px).
- Does NOT add `shadow-xl`, `shadow-lg`, `shadow-md`, `ring-1`, `ring-black/5`, or any elevation to the 3 effort-stat tiles. UI-SPEC §"3-stat top grid → Tile is FLAT — boundary enforcement (Design DNA axis 4)" + the medical-rigor boundary rule from Phase 5 CONTEXT: "the data IS the content; elevation would compete with the data per axis 4. This is the boundary where 'Airbnb-grade' stops and 'medical rigor' wins."
- Does NOT add `hover:scale`, `hover:shadow`, `hover:bg-*`, or `cursor-pointer` to the 3 effort-stat tiles. They are NOT clickable; hover-fancy lies about interactivity per Design DNA axis 5.
- Does NOT bump the 3-stat tile NUMBER typography (`text-2xl` on lines 139, 145, 151). The number stays `text-2xl` at all widths so it does not compete with the H1's `md:text-4xl`.
- Does NOT touch DrinkVoidTimeline.tsx, SummaryObservations.tsx, DaySummaryCard.tsx, or any other summary subcomponent. UI-SPEC §"DrinkVoidTimeline + SummaryObservations" + §"DaySummaryCard" verified: all 3 are already responsive-correct, all FLAT per axis 4, all using logical CSS. The `ml-2` in DaySummaryCard.tsx line 41 is a pre-existing physical-CSS leak flagged for Phase 8 visual-qa (not Phase 7).
- Does NOT touch the locked-state branch (line 83 `<h2 className="text-xl...">{t('diaryLocked')}</h2>`). The locked-state UX is intentionally restrained — celebrating a state the patient does not have yet would be wrong.
- Does NOT touch the back-link, hero icon CheckCircle2, identityFrame italic, yoursFirst italic, top-standout Sparkles card, data warning, reflection prompt, coming-back card, or Help link. UI-SPEC §"Mobile invariants per summary section" verified all are NO CHANGE.
