    ```ts
    // Dirty-state tracking — per UI-SPEC §"Reset-on-Cancel Pattern" → LogDrinkForm:
    // interacted beyond defaults if any of drinkType/volume/note/time differs from initial.
    const isDirty = useMemo(() => {
      if (drinkType !== defaultDrinkType) return true;
      if (volume !== defaultVolume) return true;
      if (note.length > 0 && note !== (editEntry?.note ?? '')) return true;
      if (time !== initialTimeSnapshotRef.current) return true;
      return false;
    }, [drinkType, volume, note, time, defaultDrinkType, defaultVolume, editEntry]);

    useEffect(() => {
      onDirtyChange?.(isDirty);
    }, [isDirty, onDirtyChange]);
    ```

    Notes:
    - `note` comparison uses `note !== (editEntry?.note ?? '')` so an existing note on edit-mode doesn't immediately flag dirty
    - For new entries (`editEntry === undefined`), `editEntry?.note ?? ''` is `''`, so the comparison is `note.length > 0 && note !== ''` → simplifies to `note.length > 0`
    - For edit entries, `editEntry?.note` is the stored note value; if the user has not typed, `note === editEntry.note`, so `isDirty` stays false based on this field
    - The `onDirtyChange` useEffect fires every time `isDirty` changes; DayPageClient (Plan 06-10) consumes it via `setActiveFormDirty(boolean)` state

    **Step 5: Add the Enter-advance / Enter-save handler on the outer wrapper (line 209):**

    Change the outer div from:
    ```
    <div className="select-none min-h-[70vh]">
    ```

    To:
    ```
    <div
      className="select-none min-h-[70vh]"
      onKeyDown={(e) => {
        const target = e.target as HTMLElement;
        const tag = target.tagName;
        // Preserve native Enter behavior inside textareas (newline) and text inputs (commit-edit).
        if (tag === 'TEXTAREA') return;
        if (tag === 'INPUT' && ((target as HTMLInputElement).type === 'text' || (target as HTMLInputElement).type === 'number')) return;
        if (e.key !== 'Enter') return;
        if (e.shiftKey) return;
        e.preventDefault();
        if (step < TOTAL_STEPS) {
          // Same gate as the sticky Next button (line 350): volume > 0
          if (volume > 0) goToStep(step + 1);
        } else {
          // Final step (2): same path as the Save button click handler
          handleSave();
        }
      }}
    >
    ```

    Behavior contract per UI-SPEC §"Form-by-Form Specs" → LogDrinkForm → "Enter-advance behavior per step":
    - Step 1 with `volume > 0`: Enter advances to Step 2
    - Step 1 with `volume <= 0`: Enter is a no-op (matches the disabled Next button state)
    - Step 2: Enter triggers `handleSave()` (which performs its own validation chain — time warnings, duplicate-minute check, etc.)
    - Enter inside the note textarea: native newline behavior preserved
    - Enter inside the TimePicker text input or VolumeInput edit input: native commit behavior preserved (VolumeInput.tsx line 55 already handles its own Enter)

    `goToStep` is defined at line 126 (`useCallback`). `handleSave` is defined at line 157 (`useCallback`). Both are stable references; the outer onKeyDown closure captures the current value via closure-over-state pattern (which React re-renders correctly on every step/volume change).
